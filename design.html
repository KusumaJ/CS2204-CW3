For the Event Handlers

I set up in HTML on

  • ticket page where seats are selected
  • index page where video should be changed
    • ps.though those are written in script, they're inside the html tag

I set up in JS in every single page using the document.addEventListener('DOMContentLoaded',..some function..) because a lot of scripts should be run after page is loaded and I don't want to put the script in the last line

For the Index page: mainly divided into 3 division

  • I use js to generate information related to the movies (data given in separated js file movies.js)
  • I use function changevid to deal with the video changes due to user'clicks

For the Now Showing

  • I use js to generate information related to the movies plus the cinemas (data given in separated js file movies.js and also cinemas.js)
  • I use js to instantly change the details of the available movies when new cinema is selected
  • Some parts of the script are also written to provide detail for the next page (via form; get method)

For the Ticket page

  • I retrieve information from the url. Then, keep and make use of it in a proper form
  • I use function clickFunc() to deal with the events that user select the seat (show info/ reserve the seat by color)
  • Some parts of the script are also written to provide detail for the next page (via form; get method)

For the Print page

  • I retrieve information from the url and make use of it in a proper form
  • I use js to generate the html of each ticket to be printed

Sketch Diagram

For the header, I use the animation to move the logo above the site name under the given condition

For the Stylesheet

I use 2 sheets: one for the whole sites [style.css] and one specifically for the printing [mediaprint.css].

Styles are grouped on every page depending on necessity as I try not to overuse the id and class selection. Detail of the whole theme is given above. Other specific detail for each page will be given below.

For the CSS3 features: position(line) mentioned are on style.css

  • Rounded corners are used several times: header image(line 27), header (line 36), #movieexample on homepage and #booklocation on nowshowing page (line 101), #confirmbook input on ticket page (line 170), submit button on nowshowing page (line 230)
  • Gradient color is also used here for the background of the division of the movie on homepage (line 121) to make the division clear and pretty but not messy

For the Home page: mainly divided into 3 division

  1. movieexample here contains videos
  2. nowshowing consists of 3 movies with image and text. I include a link to the nowshowing page as it will suggest the user to look further into detail about booking
  3. upcoming goes the same way with nowshoeing but there is no link to another page

For the Now Showing page: mainly divided into 3 division

  1. booklocation form allows user to choose location. Later on, it can be actually used with the help of javascript
  2. booknowshowing consists of 3 movies' form with timeslots to book
  3. locations contains 3 locations (map image plus address)

For the Ticket page

  1. Headings: here all information are important so I use h3 tags alone for all
  2. seatplan with table visualizes the plan of the theatre well
  3. confirmbook using form

For the Print page

Simple divs are used only to separate each ticket apart

Distinct stylesheet is used for the print purpose to hide header, footer, and other unwanted information as well as make one ticket stay in one single page
